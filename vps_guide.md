# 🚀 Руководство по деплою на VPS (Ubuntu/Debian)

Это руководство поможет вам перенести бота с Vercel на ваш собственный сервер (VPS), что обеспечит максимальную стабильность и нативный авто-запуск задач каждые 15 минут.

## 1. Подготовка сервера

Подключитесь к вашему серверу по SSH и выполните команды:

```bash
# Обновление системы
sudo apt update && sudo apt upgrade -y

# Команда для подключения (уже с вашим IP):
# ssh root@155.212.219.233

# Установка Node.js (v20+)
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# Установка PM2 (менеджер процессов)
sudo npm install -g pm2
```

## 2. Клонирование и настройка

1. Загрузите файлы проекта на сервер (через git или SFTP).
2. Перейдите в папку проекта:
   ```bash
   cd wb_feedback
   npm install
   ```
3. Создайте файл `.env` (скопируйте его содержимое из локального проекта или Vercel).
4. Убедитесь, что `APP_URL` в `.env` указывает на ваш домен или IP с HTTPS.

## 3. Запуск через PM2

Для запуска используйте подготовленный файл конфигурации:

```bash
pm2 start ecosystem.config.js
pm2 save
pm2 startup
```

## 4. Настройка HTTPS (Nginx + Certbot)

Telegram требует HTTPS для работы вебхуков.

1. Установите Nginx: `sudo apt install nginx`
2. Настройте конфигурацию (проброс на порт 3000).
3. Установите SSL через Certbot:
   ```bash
   sudo apt install certbot python3-certbot-nginx
   sudo certbot --nginx -d ваш-домен.ru
   ```

## 5. Обновление Вебхука

После запуска сервера на VPS, откройте в браузере:
`https://ваш-домен.ru/api/setup`

Это привяжет бота к новому адресу вашего сервера.
